2. Provider Decision (Big Architectural Choice)

Path A: Twilio production WhatsApp
  • ✅ We already have a Twilio account + working webhook code.
  • ✅ Twilio handles Meta liaison, template submission, error handling.
  • ❌ Higher per-message cost (~AUD $0.07–$0.12 per conversation vs Meta's ~$0.04).
  • ❌ Limited control over advanced WhatsApp features (catalogue, flows).
Path B: Direct Meta Cloud API
  • ✅ Cheapest at scale.
  • ✅ Full access to WhatsApp Flows, Catalogue, Interactive messages, Click-to-WhatsApp ads.
  • ❌ More setup work — we manage tokens, webhooks, template submissions ourselves.
  • ❌ Stricter rate limits early on (1,000 conversations/day until verified).
Path C: 360dialog / MessageBird (BSP)
  • ✅ Good middle ground — flat monthly fee, no per-message markup.
  • ❌ Yet another vendor + contract.

⚠️ Edge Cases, Bottlenecks & Complications

  1. The 24-hour customer service window. Once a customer messages you, you can reply freely for 24h. After that, every outbound message MUST be a pre-approved template. Forgetting this kills automation flows.
  2. Template approval delays. Each template needs Meta approval, usually 24–72 hours. New product launches / sales need templates submitted well in advance. Rejection reasons can be vague ("too promotional"). Plan a template library.
  3. Template categories & pricing. Marketing > Utility > Authentication in cost. Mis-categorising leads to rejection or higher bills. Australia is a Tier-1 market = highest pricing tier.
  4. Quality rating. Meta tracks user complaints. Too many "report spam" / blocks drops your number to Medium → Low → Flagged → permanent ban. Always offer easy opt-out.
  5. Number portability is painful. Choose the WhatsApp number carefully — switching providers later (Twilio ↔ Meta direct) is multi-day downtime.
  6. One number = one WABA. Cannot share a number across Twilio + Meta direct. If we ever want to migrate, must "release" then "re-register".
  7. Voice notes from customers. Customers love sending voice notes. Need STT pipeline (ElevenLabs scribe or OpenAI Whisper) to feed text into the agent.
  8. Image messages. Customers send paint colour photos or pictures of damaged walls. Need a vision model (GPT-4o, Claude) to interpret. Adds cost per image.
  9. Latency. WhatsApp users expect responses within seconds. Each message hop is: Meta → Vercel webhook → ElevenLabs API → Vercel → Meta. Cold starts on Vercel can push this to 5–10s. May need warm functions or edge runtime.
  10. Concurrent conversations. 50 customers messaging at once. Need a session store keyed by their phone number, not a single global state.
  11. Out-of-hours scenarios. When a human handoff is needed but the team is asleep. Define an explicit "we're closed, leave a message" template + Shopify flag.
  12. Misdirected messages. People send "hi" / "test" / wrong number. Need an idle/noise filter so you don't burn template credits replying to noise.
  13. Phone-only customers. Many WhatsApp leads will never give email. Shopify customer record will be phone-only — but Shopify customers historically need email or phone (now either works). Decide what's the minimum to count as a "lead".
  14. Meta business verification rejections. Common reasons: business address mismatch, ABN not visible on documents, website not linked. Plan time for re-submissions.
  15. Opt-in compliance. Australian Spam Act + Meta policy: outbound to a number requires explicit opt-in (e.g., checkbox at checkout, or the customer messaged us first). Cannot import contact lists and blast.
  16. Meta API versioning. Cloud API breaks at version bumps (~yearly). Need to monitor deprecation notices.
  17. Pixel + CAPI dual-tracking deduplication. When firing events both client-side (Pixel) and server-side (CAPI), missing the event_id on both sides causes double-counting in Ads Manager.
  18. iOS/Android quirks. Interactive buttons render differently. Test on both. Long messages get clipped on older Androids.
  19. Catalogue sync (if used). Meta Catalogue must be kept in sync with Shopify products. Either via Meta's Shopify integration app or a custom feed. Stale prices = customer complaints.
  20. Cost spiral. Each conversation triggers: WhatsApp API charge + ElevenLabs text API + Shopify API + (optional) vision/STT. Easy to hit $0.20–$0.50 per conversation if not careful.
  21. Privacy / GDPR / Australian Privacy Act. Storing transcripts of WhatsApp conversations needs a clear privacy policy update.

❓ Questions to Clarify Before Building

Q1. Provider. Twilio (faster start, more expensive), Meta Cloud API direct (cheapest, more work), or 360dialog/MessageBird (BSP middle ground)? My recommendation: Meta Cloud API direct for long-term cost + feature access.
Q2. WhatsApp phone number. New dedicated number, or one of your existing numbers? Important: it cannot have a personal WhatsApp registered. The existing Twilio sandbox number won't carry over to production.
Q3. Meta Business verification. Has Paint Access already done Meta Business verification (the document upload step)? If not, this is the longest blocker — can take a week.
Q4. Funnel scope. Pure support + Q&A, or full sales (recommend → quote → payment link → close)? Recommendation: start with support + lead-capture, add the payment-link funnel in Phase 2 once approved.
Q5. Voice notes / images. Should the bot understand voice notes and product photos? Adds STT + vision-model cost (~$0.01–$0.05 each).
Q6. Inbound entry points. Where do customers find the WhatsApp number? Storefront widget? Click-to-WhatsApp ads on Facebook/Instagram? Both? Each requires different setup.
Q7. Languages. English only, or also Russian / other? WhatsApp templates need to be approved per language.
Q8. Out-of-hours behaviour. Should the bot pretend to be 24/7, or honest ("we're closed, I'll log this and someone will respond tomorrow")?
Q9. Human handoff trigger. When the bot can't help, do you want a Slack/email notification to the team? Or a "request a human" template button?
Q10. Marketing opt-in. When someone first messages, do we auto-add them to Omnisend marketing list, or only after explicit "yes, send me deals"?
Q11. Conversation log retention. How long to keep WhatsApp transcripts? 90 days? 2 years? Forever?
Q12. Meta Pixel. Is the Pixel currently installed on paintaccess.com.au? Is it firing correctly? Need to audit before CAPI work.
Q13. Catalogue. Use Meta Commerce Catalogue (lets WhatsApp show rich product cards), or just text + links?
Q14. Test plan. Who tests during sandbox/staging? I'll need at least 2 test phone numbers added to the WABA early on.
Q15. Budget ceiling. What's the monthly WhatsApp messaging budget? Helps decide template strategy and provider.

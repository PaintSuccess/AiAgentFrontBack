🎯 What You're Asking For

Every meaningful AI agent conversation (voice or chat) should:

  1. Collect contact info — first name, last name, email, optional phone.
  2. Ask if they're an existing customer + consent to be added to the database.
  3. Push into Shopify Customers via API (create or update), tagged AI Agent.
  4. Log the journey — what they asked, what they were interested in, did they buy.
  5. Feed downstream marketing — Omnisend / Shopify Flow for email, SMS, retargeting.

⚠️ Edge Cases, Bottlenecks & Complications

  1. Voice agents can't reliably repeat back emails. STT confuses "j"/"g", "five"/"nine", domains like gmail vs g-mail. Need an explicit confirmation step: "So that's john at gmail dot com, J-O-H-N — correct?"
  2. Duplicate customers. Shopify allows multiple customers with the same phone but unique emails. Logic must be: lookup by email → if found, update tags/notes; if not, create new.
  3. Anonymous voice calls (Twilio inbound). No customer context. Agent must collect everything from scratch.
  4. Consent & compliance. Australian Spam Act 2003 + GDPR (if any EU traffic). Must record explicit opt-in for marketing. "AI Agent" tag alone is not legal consent.
  5. Tool-call latency mid-conversation. Calling save_lead mid-call adds a 1–3 second pause. Better strategy: gather data in memory, push everything in a single post-call webhook at the end.
  6. Transcript size limits. ElevenLabs transcripts can be 5–50KB. Shopify customer notes have a 5,000-char limit; metafields can hold more (up to 64KB) but need a defined namespace.
  7. PII in logs. Vercel logs will contain emails, phones, transcripts. Retention policy / scrubbing may be required.
  8. Idempotency. If the post-call webhook fires twice (ElevenLabs retries), duplicate notes will be created. Must dedupe by conversation_id.
  9. Existing-customer flag mismatch. Customer says "yes, I'm a customer" but email isn't in DB. Agent shouldn't argue — just collect and tag.
  10. Tag explosion. One tag per interest topic leads to dozens of tags. Recommendation: single AI Agent tag + structured JSON metafield for interests.
  11. Cost creep. Every conversation = Shopify Admin API calls + ElevenLabs minutes. Bulk testing could burn through quota again.
  12. Widget form vs voice agent overlap. If a user submits the callback form AND talks to the agent, we'll double-create the lead. Email must be the single source-of-truth deduplication key.
  13. Phone normalisation. 0488 826 453 vs +61 488 826 453 vs 61488826453 — must normalise to E.164 before storing.

📋 Phased Implementation Plan

Phase 1 — Shopify Access + Backend Foundation
  • Add write_customers and read_customers scopes to Shopify custom app (owner does this, then rotates token).
  • New endpoint: POST /api/shopify/customer/upsert.
  • New endpoint: POST /api/agent/log-conversation (handles ElevenLabs post-call webhook).
Phase 2 — Agent Rules + New Tool
  • Add save_lead server tool to ElevenLabs agent.
  • Update system prompt with lead-capture flow, opt-in language, refusal handling.
  • Configure ElevenLabs post-call webhook URL to point at /api/agent/log-conversation.
Phase 3 — Storefront Widget
  • Pass UTM, referrer, and current page URL into agent dynamic variables.
  • Pass logged-in customer's last_name and phone when available.
  • Store collected email in localStorage for cross-session continuity.
Phase 4 — Omnisend
  • Decide: native Shopify→Omnisend sync vs direct API push.
  • Define segments matching tags (AI Agent, AI Lead).
  • If direct: add /api/omnisend/upsert-contact endpoint.
Phase 5 — Polish & Compliance
  • Consent recording (spoken or checkbox).
  • PII redaction in Vercel logs.
  • Deduplicate webhook calls by conversation_id.
  • Phone normalisation to E.164.

❓ Questions to Clarify Before Building

Q1. Shopify scopes. Can you add write_customers + read_customers to the existing Shopify custom app yourself, or do you need a step-by-step walkthrough?
Q2. Marketing consent. Should the agent explicitly ask "May I add you to our list to receive emails about paint deals and tips?" and only set the accepts_marketing flag if they say yes? Or are you OK with a softer transactional-only approach?
Q3. Existing-customer detection. When the customer says "yes, I'm already a customer", should the agent confirm by email lookup, or just trust them and add the tag?
Q4. Tags vs metafields. Just a single AI Agent tag, or also per-topic tags like AI Lead - Sprayers? Recommendation: single tag + structured JSON metafield.
Q5. Transcript storage. Full transcript in a Shopify metafield (visible to staff), or only a short AI-generated summary — with full transcript staying in ElevenLabs?
Q6. Omnisend strategy. Native Shopify→Omnisend sync (simpler, minutes lag) or direct API push (real-time)? If direct, please provide an Omnisend API key.
Q7. Language. Should Jessica ask for lead details in English only? The task was written in Russian — should the agent also support Russian-speaking customers?
Q8. What counts as a "useful" conversation? Save every call as a lead (including 10-second hi/bye), or only when at least an email was collected? Recommendation: only save when email is present.
Q9. Anonymous Twilio inbound calls. These have a phone number but no email. Should they also create Shopify customers (phone-only), or skip CRM logging unless email is collected?
Q10. Tagging existing customers. If the contact already exists in Shopify, should we add the AI Agent tag to mark the new AI interaction, or leave existing customers untagged?

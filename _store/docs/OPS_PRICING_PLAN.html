AI Automation for Operations & Pricing — Implementation Plan

Paint Access — Future Feature #3 — Planning Document v1.0

Reading order: This is the third plan in the series. Plan #1 = LEAD_CAPTURE_PLAN.html (customer-facing AI funnel). Plan #2 = WHATSAPP_META_PLAN.html (WhatsApp channel). This plan #3 is internal/back-office — it shares no user-facing surface with #1 or #2, but it does share the same Shopify store, the same Vercel backend, and the same data pipeline.

1. What Was Asked

Two distinct but related back-office automation blocks:

Block A — Operations

  • AI handling of orders (status sync, exceptions, supplier dispatch).
  • AI handling of suppliers (Oldfields, Uni-Pro, Equi, Pulse, etc.).
  • Inventory management — central truth of stock across own warehouse + suppliers.
  • Available-stock sync on the website (Shopify) — what customers see must reflect what can actually ship.
  • Connecting supplier feeds / APIs — pulling product catalogues, stock levels, prices, lead times.

Block B — Pricing Intelligence

  • Parsing competitor prices (web scraping or feeds).
  • Price analytics — historical trends, margin impact, elasticity hints.
  • Pricing recommendations — AI suggests when to raise/lower.
  • Automatic tables & dashboards.
  • AI hints on price changes & margin — explanations, not just numbers.

3. What Is Missing

3.1 Shopify Permissions

Current scopes are read-only for inventory and products. We need:

  • write_inventory — push supplier stock into Shopify
  • write_products — update prices & cost of goods
  • write_orders — mark fulfillments, attach supplier reference
  • read_locations + write_locations — if we model each supplier as a virtual location
  • read_price_rules + write_price_rules — automated promo creation (optional)

3.2 Supplier Connectivity

This is the single biggest unknown. Each supplier exposes data differently. Possible formats per supplier:
  • REST API with auth (best case)
  • Daily FTP/SFTP CSV drop
  • Emailed Excel file (worst case — needs parsing)
  • EDI feed (rare for paint trade)
  • No feed at all — only PDF price lists (requires OCR or manual upload)

3.4 Data Store

Shopify alone is not enough for this plan. We need a database for:

  • Supplier catalogues (SKU mapping, cost, lead time, MOQ)
  • Competitor price history (time-series — Shopify can't store this)
  • Price-change audit log (who/what/when/why)
  • Job run history (which sync ran when, what changed, errors)

Recommended: Supabase Postgres (free tier sufficient for start; same project can host plan #1 data too).

3.5 AI Layer

  • Operations AI — uses GPT-4o-mini or Claude Haiku to classify supplier emails, normalise product names across catalogues, match SKUs to Shopify variants. Cheap, batched.
  • Pricing AI — uses GPT-4o or Claude Sonnet to produce reasoned recommendations ("raise SKU X by 4% — competitor Y dropped, our margin is 38%, demand stable last 30 days").

3.6 Internal UI / Dashboard

This work needs a place for you to review and approve AI suggestions — autopilot on prices is too risky early on.

  • Option A (fastest): Google Sheets synced via cron — simple, you already know it.
  • Option B (mid): Retool / Appsmith — drag-and-drop admin UI, ~$10/user/month.
  • Option C (custom): Lightweight React dashboard on Vercel — most flexible, more dev time.

3.7 Competitor Price Scraping Infrastructure

  • List of competitor URLs to monitor (per-SKU mapping).
  • Scraping engine — Playwright on Vercel cron (works for ~80% of sites) or ScrapingBee/Bright Data proxy ($30–$100/mo for sites with anti-bot).
  • Legal review — scraping prices is broadly OK in AU, but ToS violations can lead to IP bans. Robots.txt should be respected.

4. Edge Cases & Failure Modes

  1. Supplier feed late/missing — keep last-known-good stock for X hours, then mark items as "stock TBC" rather than zero out.
  2. Supplier renames SKU — fuzzy match on product name + size + colour, surface for manual approval.
  3. Same product, multiple suppliers — which one wins? Rule: lowest landed cost, or fastest lead time, or per-product preference.
  4. Supplier oversells — they report stock, we sell, they're actually out. Need exception queue + email/SMS alert to you.
  5. Customer order arrives during supplier sync — race condition. Use Shopify's atomic InventoryAdjustQuantity, never set absolute values.
  6. Shopify variant doesn't exist for a supplier SKU — auto-create draft product for your review (never auto-publish).
  7. Bulk price update accidentally tanks margin — every price change goes through a "guard rail" check: max ±15% from current, never below cost × 1.10.
  8. Competitor scraping breaks (DOM changed) — graceful fail, alert you, don't feed bad data into recommendations.
  9. Competitor shows "Call for price" — don't treat as $0. Mark as unknown, skip in calculations.
  10. Currency mismatch — supplier feed in USD/EUR, store in AUD. Need FX rate + buffer.
  11. GST handling — supplier costs ex-GST, retail inc-GST. Pricing engine must keep these straight or margins look wrong by 10%.
  12. Discontinued line — supplier drops product. Don't just delete — mark as "supplier-discontinued", keep selling existing stock, then archive.
  13. Backorders — supplier has 3-week lead time. Should the product be hidden, marked "pre-order", or kept normal? Per-product rule needed.
  14. Manual override stomped by next sync — if you manually set a price, the sync must respect a "locked" flag.
  15. Shopify rate limits — bulk updates can hit 40 req/sec ceiling. Need queue with backoff.
  16. Pricing rec creates a loop — we lower price, competitor lowers, we lower again. Damping rule: max 1 change per SKU per 7 days.
  17. Promo periods — Black Friday, EOFY. AI must know to suspend normal pricing logic during marked windows.
  18. B2B vs retail pricing — Shopify B2B is separate. Plan must decide if it's in scope.
  19. Cost change without retail change — supplier raises cost 8%. Do we absorb or pass on? Per-category rule.
  20. Audit / compliance — ACCC rules on misleading pricing (e.g. "was $X" claims). Logged history is mandatory if we ever display strikethrough.
  21. Multi-location stock — if PaintAccess has more than one Shopify location, sync must target the right one.
  22. Supplier credentials leak — all in Vercel env vars, never logged, never echoed in errors.

6. Phased Rollout

Phase 0 — Discovery (no code)

  • Inventory all suppliers, document feed format for each.
  • Inventory all competitors to monitor, get URL list.
  • Decide dashboard route (Sheets / Retool / custom).
  • Decide Supabase vs alternative (Neon, Planetscale).
  • Add required Shopify scopes; rotate token.

Phase 1 — Foundation

  • Set up Supabase project + schema (suppliers, products_map, price_history, audit_log, job_runs).
  • Build supplier connector framework — one adapter per format (API/CSV/Excel).
  • Build first supplier integration end-to-end (pick easiest one as proof).
  • Manual review UI — Sheets with daily export.

Phase 2 — Inventory Sync Live

  • All suppliers connected.
  • Hourly merge + push to Shopify with guard rails.
  • Exception queue with Slack/email alerts.
  • You operate in monitor mode — system suggests, you approve, system writes.

Phase 3 — Order Routing

  • Shopify webhook → backend → choose supplier → email/API to supplier → mark fulfilled.
  • Customer-facing tracking unchanged (Shopify shows progress).

Phase 4 — Competitor Pricing

  • Scraper running on cron.
  • Daily AI report: "12 SKUs flagged, 3 strong raise, 5 hold, 4 watch".
  • One-click approve/reject in dashboard.

Phase 5 — Pricing Autopilot (optional)

  • Once trust is established, enable auto-apply for low-risk categories (e.g. ±3% within guard rails).
  • High-margin or strategic SKUs stay manual forever.

Phase 6 — Analytics

  • Margin trend dashboards.
  • Supplier scorecard (fill rate, lead time, accuracy).
  • Price elasticity hints from sales data.

11. Recommendation

Best sequencing:
  1. Spend 1 week on Phase 0 discovery — supplier audit. This unblocks everything.
  2. Build Phase 1 + Phase 2 (inventory sync only) as proof. Pricing comes later.
  3. Keep humans in the loop for all price writes for at least 60 days.
  4. Add Phase 4 (competitor pricing) in parallel with Phase 2 — it's read-only and won't break anything if it fails.
This plan has the highest ROI of the three because it touches every order and every margin dollar. But it also has the highest integration risk because of supplier variability. Phase 0 is non-negotiable.
Hard "do not start" checklist:
  • Do not start Phase 1 without sample supplier feeds in hand.
  • Do not enable any auto-write to Shopify without guard rails coded and tested.
  • Do not scrape competitors without robots.txt check + rate limiting.
  • Do not store supplier credentials anywhere outside Vercel env vars.
